Get cached or create new iconv descriptor for encoding conversion.

Parameters
fromcodeSource encoding name
tocodeTarget encoding name
Returns
Shared pointer to iconv descriptor with automatic cleanup

This function manages the iconv descriptor cache, providing thread-safe access to conversion descriptors. If a descriptor for the requested encoding pair exists in cache, it returns the cached version. Otherwise, it creates a new descriptor and adds it to the cache.

Caching Benefits:
  • Reduces overhead of repeated iconv_open() calls
  • Improves performance for repeated conversions
  • Automatic cleanup via RAII when no longer needed
  • Thread-safe access with mutex protection
Error Handling:
Returns nullptr if iconv_open() fails (invalid encoding names)

Get human-readable error description for iconv error codes.

Parameters
err_codeThe iconv error code to translate
Returns
Human-readable error message string

This static function provides standardized error messages for iconv error codes. It maps common errno values that iconv functions return to descriptive text.

Common Error Codes:
  • EINVAL: Invalid encoding names or unsupported conversion
  • EILSEQ: Invalid byte sequence in input
  • E2BIG: Output buffer too small
  • ENOMEM: Memory allocation failure
Since
1.0.0.1
Parameters
err_codeThe errno value returned by iconv operations
Returns
Human-readable error description string

Translates iconv-specific error codes into descriptive messages that can be displayed to users or used for debugging purposes.

Supported Error Codes:
  • EILSEQ: Invalid multibyte sequence encountered
  • EINVAL: Incomplete multibyte sequence at end of input
  • E2BIG: Output buffer too small for conversion result
  • EBADF: Invalid conversion descriptor (encoding not supported)
Usage Example:
auto result = UniConv::ConvertEncoding(input, "UTF-8", "GBK");
if (!result.IsSuccess()) {
std::string error_desc = UniConv::GetIconvErrorString(result.error_code);
std::cerr << "Conversion failed: " << error_desc << std::endl;
}
