This is the core conversion function that handles conversion between any two supported encodings using the iconv library. It provides comprehensive error handling and memory management.

Supported Encodings:
  • UTF-8, UTF-16LE, UTF-16BE
  • GBK, GB2312, BIG5
  • Windows code pages (CP936, CP950, etc.)
  • ISO-8859 series
  • And many others supported by iconv
Error Handling:
  • Returns error code and message on failure
  • Handles invalid input sequences gracefully
  • Provides detailed error information

Get the current system's default character encoding.

Returns
String representing the current system encoding name
Return values
UTF-8if the system uses UTF-8 encoding
GBKif the system uses GBK encoding (Windows CP936)
BIG5if the system uses Big5 encoding (Windows CP950)
SHIFT_JISif the system uses Shift_JIS encoding (Windows CP932)
CP####for other Windows code pages

This function detects the system's active code page on Windows using GetACP() and returns the corresponding encoding name. On Linux systems, it typically returns "UTF-8".

Platform Support:
  • Windows: Uses GetACP() to detect active code page
  • Linux: Returns "UTF-8" by default

This function detects the system's active code page (Windows) or assumes UTF-8 (Unix-like systems) and returns the corresponding encoding name that can be used with iconv.

Platform Support:
  • Windows: Uses GetACP() to detect active code page
  • Unix/Linux: Returns "UTF-8" as default
Common Return Values:
  • "GBK" (CP936) - Simplified Chinese
  • "BIG5" (CP950) - Traditional Chinese
  • "SHIFT_JIS" (CP932) - Japanese
  • "UTF-8" (CP65001) - Unicode UTF-8
  • "CP####" - Generic code page format
