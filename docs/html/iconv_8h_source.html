44/* We would like to #include any system header file which could define
45 iconv_t, 1. in order to eliminate the risk that the user gets compilation
46 errors because some other system header file includes /usr/include/iconv.h
47 which defines iconv_t or declares iconv after this file, 2. when compiling
48 for LIBICONV_PLUG, we need the proper iconv_t type in order to produce
49 binary compatible code.
50 But gcc's #include_next is not portable. Thus, once libiconv's iconv.h
51 has been installed in /usr/local/include, there is no way any more to
52 include the original /usr/include/iconv.h. We simply have to get away
53 without it.
54 Ad 1. The risk that a system header file does
55 #include "iconv.h" or #include_next "iconv.h"
56 is small. They all do #include <iconv.h>.
57 Ad 2. The iconv_t type is a pointer type in all cases I have seen. (It
58 has to be a scalar type because (iconv_t)(-1) is a possible return value
59 from iconv_open().) */
60
