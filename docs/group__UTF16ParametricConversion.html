This overload allows conversion of UTF-16LE data with explicit length specification, useful when working with non-null-terminated buffers or when the exact length is known.

Safety Notes:
  • Input validation: Returns empty string if input is null or len is 0
  • Memory safety: Uses reinterpret_cast with known buffer size
  • Error handling: Returns empty string on conversion failure
