\doxylink{iconv_8h_a44c0f9426dd91c55d783618e535404b7}{iconv()}{\ttfamily 函数将一个字符编码的字符序列转换为另一个字符编码的字符序列。}cd\`{} 参数是一个转换描述符

，由先前对 {\ttfamily \doxylink{iconv_8h_abc93f77b3b8b5891765c9f3c483ec013}{iconv\+\_\+open(3)}} 的调用创建；该转换描述符定义了 {\ttfamily \doxylink{iconv_8h_a44c0f9426dd91c55d783618e535404b7}{iconv()}} 在转换过程中使用的字符编码。{\ttfamily inbuf} 参数是一个变量的地址，该变量指向输入序列的第一个字符；

{\ttfamily inbytesleft} 表示该缓冲区中的字节数。

{\ttfamily outbuf} 参数是一个变量的地址，该变量指向输出缓冲区中可用的第一个字节；

{\ttfamily outbytesleft} 表示输出缓冲区中可用的字节数。

主要的情况是当 inbuf 不为 NULL 且 \texorpdfstring{$\ast$}{*}inbuf 不为 NULL 时。 在这种情况下，iconv() 函数会将从 \texorpdfstring{$\ast$}{*}inbuf 开始的多字节序列转换为从 \texorpdfstring{$\ast$}{*}outbuf 开始的多字节序列。 最多会读取从 \texorpdfstring{$\ast$}{*}inbuf 开始的 \texorpdfstring{$\ast$}{*}inbytesleft 个字节。最多会写入从 \texorpdfstring{$\ast$}{*}outbuf 开始的 \texorpdfstring{$\ast$}{*}outbytesleft 个字节。

{\ttfamily \doxylink{iconv_8h_a44c0f9426dd91c55d783618e535404b7}{iconv()}} 函数一次转换一个多字节字符，对于每个字符转换，它会将 {\ttfamily \texorpdfstring{$\ast$}{*}inbuf} 增加，并将 {\ttfamily \texorpdfstring{$\ast$}{*}inbytesleft} 减去已转换的输入字节数，将 {\ttfamily \texorpdfstring{$\ast$}{*}outbuf} 增加，并将 {\ttfamily \texorpdfstring{$\ast$}{*}outbytesleft} 减去已转换的输出字节数，同时更新 {\ttfamily cd} 中包含的转换状态。如果输入的字符编码是有状态的，{\ttfamily \doxylink{iconv_8h_a44c0f9426dd91c55d783618e535404b7}{iconv()}} 函数还可以将输入字节序列转换为转换状态的更新而不产生任何输出字节；这种输入称为移位序列。转换可以因五种原因停止：


\begin{DoxyItemize}
\item 输入中遇到无效的多字节序列。在本例中，它将errno设置为\+EILSEQ并返回(size\+\_\+t) -\/1。\texorpdfstring{$\ast$}{*}inbuf左指向无效多字节序列的开头。
\item 遇到一个多字节序列，该序列有效但无法转换为输出的字符编码。这种情况取决于实现和转换描述符。在\+GNU C库和\+GNU libiconv中，如果{\ttfamily cd}创建时没有使用后缀{\ttfamily //\+TRANSLIT}或{\ttfamily //\+IGNORE}，则转换是严格的：有损转换会产生这种情况。如果指定了后缀{\ttfamily //\+TRANSLIT}，则在某些情况下可以通过音译避免这种情况。在musl C库中，这种情况不会发生，因为会使用转换为`'\texorpdfstring{$\ast$}{*}\textquotesingle{}{\ttfamily 作为回退。在\+Free\+BSD、\+Net\+BSD和\+Solaris的}\doxylink{iconv_8h_a44c0f9426dd91c55d783618e535404b7}{iconv()}{\ttfamily 实现中，这种情况也不会发生，因为它们会使用转换为}\textquotesingle{}?\textquotesingle{}{\ttfamily 作为回退。当满足此条件时，}\doxylink{iconv_8h_a44c0f9426dd91c55d783618e535404b7}{iconv()}{\ttfamily 会将}errno{\ttfamily 设置为}EILSEQ{\ttfamily 并返回}(size\+\_\+t) -\/1{\ttfamily 。}\texorpdfstring{$\ast$}{*}inbuf{\ttfamily 会指向无法转换的多字节序列的开头。}
\item {\ttfamily 输入的字节序列已经完全转换，即}\texorpdfstring{$\ast$}{*}inbytesleft{\ttfamily 已减少到0。在这种情况下，}\doxylink{iconv_8h_a44c0f9426dd91c55d783618e535404b7}{iconv()}{\ttfamily 会返回在此次调用期间执行的非可逆转换的次数。}
\item {\ttfamily 在输入中遇到了一个不完整的多字节序列，并且输入字节序列在其后终止。在这种情况下，}\doxylink{iconv_8h_a44c0f9426dd91c55d783618e535404b7}{iconv()}{\ttfamily 会将}errno{\ttfamily 设置为}EINVAL{\ttfamily 并返回}(size\+\_\+t) -\/1{\ttfamily 。}\texorpdfstring{$\ast$}{*}inbuf\`{}会指向不完整多字节序列的开头。
\item 输出缓冲区没有足够的空间来容纳下一个转换后的字符。在这种情况下，{\ttfamily \doxylink{iconv_8h_a44c0f9426dd91c55d783618e535404b7}{iconv()}}会将{\ttfamily errno}设置为{\ttfamily E2\+BIG}并返回$<$tt$>$(size\+\_\+t) -\/1
\end{DoxyItemize}


\begin{DoxyItemize}
\item 另一种情况是当{\ttfamily inbuf}为{\ttfamily NULL}或{\ttfamily \texorpdfstring{$\ast$}{*}inbuf}为{\ttfamily NULL}，但{\ttfamily outbuf}不为{\ttfamily NULL}且{\ttfamily \texorpdfstring{$\ast$}{*}outbuf}不为{\ttfamily NULL}时。在这种情况下，{\ttfamily \doxylink{iconv_8h_a44c0f9426dd91c55d783618e535404b7}{iconv()}}函数会尝试将{\ttfamily cd}的转换状态设置为初始状态，并在{\ttfamily \texorpdfstring{$\ast$}{*}outbuf}处存储相应的移位序列。从{\ttfamily \texorpdfstring{$\ast$}{*}outbuf}开始，最多会写入{\ttfamily \texorpdfstring{$\ast$}{*}outbytesleft}字节。如果输出缓冲区没有足够的空间来存储此重置序列，它会将{\ttfamily errno}设置为{\ttfamily E2\+BIG}并返回$<$tt$>$(size\+\_\+t) -\/1
\end{DoxyItemize}。否则，它会根据写入的字节数增加{\ttfamily \texorpdfstring{$\ast$}{*}outbuf}并减少{\ttfamily \texorpdfstring{$\ast$}{*}outbytesleft}。

第三种情况是当{\ttfamily inbuf}为{\ttfamily NULL}或{\ttfamily \texorpdfstring{$\ast$}{*}inbuf}为{\ttfamily NULL}，且{\ttfamily outbuf}为{\ttfamily NULL}或{\ttfamily \texorpdfstring{$\ast$}{*}outbuf}为{\ttfamily NULL}时。在这种情况下，{\ttfamily \doxylink{iconv_8h_a44c0f9426dd91c55d783618e535404b7}{iconv()}}函数会将{\ttfamily cd}的转换状态设置为初始状态。


\begin{DoxyItemize}
\item {\ttfamily \doxylink{iconv_8h_a44c0f9426dd91c55d783618e535404b7}{iconv()}}函数返回在此次调用期间以不可逆方式转换的字符数量；可逆转换不会被计数。如果发生错误，{\ttfamily \doxylink{iconv_8h_a44c0f9426dd91c55d783618e535404b7}{iconv()}}会返回$<$tt$>$(size\+\_\+t) -\/1

{\bfseries{EILSEQ}} 在输入中遇到了无效的多字节序列。

{\bfseries{ENOMEM}}

在每次调用{\ttfamily \doxylink{iconv_8h_a44c0f9426dd91c55d783618e535404b7}{iconv()}}的系列操作中，最后一次调用应将{\ttfamily inbuf}或{\ttfamily \texorpdfstring{$\ast$}{*}inbuf}设置为{\ttfamily NULL}，以便刷新任何部分转换的输入。

尽管{\ttfamily inbuf}和{\ttfamily outbuf}的类型为{\ttfamily char \texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}}，但这并不意味着它们所指向的对象可以被解释为\+C字符串或字符数组：字符字节序列的解释由转换函数内部处理。在某些编码中，零字节可能是多字节字符的有效部分。

{\ttfamily \doxylink{iconv_8h_a44c0f9426dd91c55d783618e535404b7}{iconv()}}的调用者必须确保传递给函数的指针适合访问相应字符集中的字符。这包括在对齐要求严格的平台上确保正确的对齐。
